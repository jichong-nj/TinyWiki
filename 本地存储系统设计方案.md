# TinyWiki 本地存储系统设计方案

## 1. 整体设计思路

基于文件MD5哈希值构建两级目录结构，实现高效的文件存储和检索，支持原始文件、转换后文件、附件等多种类型的存储。

## 3. 目录结构说明

### 3.1 两级目录策略

使用MD5哈希值的前4位创建两级子目录：
- 一级目录：MD5前2位（如 `a1/`）
- 二级目录：MD5第3-4位（如 `b2/`）
- 最终目录：完整MD5（如 `a1b2c3.../`）

**优点**：
- 避免单个目录下文件过多（文件系统性能问题）
- 支持快速定位文件
- 分布式存储友好

### 3.2 存储类型分类

| 目录 | 用途 | 说明 |
|------|------|------|
| `original/` | 原始文件 | 用户上传的原始文件 |
| `converted/` | 转换文件 | PDF、HTML、TXT等格式 |
| `chunks/` | 文档片段 | 用于RAG的文本片段 |
| `attachments/` | 附件 | 文档中的图片、附件等 |
| `embeddings/` | 向量嵌入 | 语义搜索向量 |
| `thumbnails/` | 缩略图 | 文档预览缩略图 |

## 4. 数据库模型设计

### 4.1 FileStorage 模型

```python
from django.db import models
import hashlib

class FileStorage(models.Model):
    """文件存储模型"""
    STORAGE_TYPES = [
        ('original', '原始文件'),
        ('converted', '转换文件'),
        ('attachment', '附件'),
        ('chunk', '文档片段'),
        ('embedding', '向量嵌入'),
        ('thumbnail', '缩略图'),
    ]
    
    id = models.BigAutoField(primary_key=True)
    md5_hash = models.CharField(max_length=32, unique=True, db_index=True, verbose_name='MD5哈希')
    file_name = models.CharField(max_length=255, verbose_name='文件名')
    file_size = models.BigIntegerField(verbose_name='文件大小(字节)')
    file_type = models.CharField(max_length=50, verbose_name='文件类型')
    mime_type = models.CharField(max_length=100, blank=True, verbose_name='MIME类型')
    storage_type = models.CharField(max_length=20, choices=STORAGE_TYPES, verbose_name='存储类型')
    relative_path = models.CharField(max_length=500, verbose_name='相对路径')
    created_at = models.DateTimeField(auto_now_add=True, verbose_name='创建时间')
    updated_at = models.DateTimeField(auto_now=True, verbose_name='更新时间')
    
    # 关联文档（可选）
    document = models.ForeignKey('documents.Document', on_delete=models.SET_NULL, 
                               null=True, blank=True, related_name='files', 
                               verbose_name='关联文档')
    
    class Meta:
        db_table = 'file_storage'
        verbose_name = '文件存储'
        verbose_name_plural = verbose_name
    
    def __str__(self):
        return f"{self.file_name} ({self.md5_hash})"
    
    @property
    def full_path(self):
        """获取完整文件路径"""
        from django.conf import settings
        return settings.STORAGE_ROOT / self.relative_path
    
    @classmethod
    def get_md5(cls, file_obj):
        """计算文件MD5"""
        md5 = hashlib.md5()
        for chunk in file_obj.chunks():
            md5.update(chunk)
        return md5.hexdigest()
    
    @classmethod
    def get_storage_path(cls, md5_hash, sub_dir=''):
        """根据MD5获取存储路径"""
        level1 = md5_hash[:2]
        level2 = md5_hash[2:4]
        path_parts = [level1, level2, md5_hash]
        if sub_dir:
            path_parts.insert(0, sub_dir)
        return '/'.join(path_parts)
```

## 6. 文件存储服务类

### 6.1 StorageService

```python
import os
import hashlib
from pathlib import Path
from django.conf import settings
from .models import FileStorage

class StorageService:
    """文件存储服务"""
    
    @classmethod
    def save_file(cls, file_obj, storage_type='original', document=None, 
                 original_name=None):
        """
        保存文件
        
        Args:
            file_obj: 文件对象
            storage_type: 存储类型
            document: 关联文档
            original_name: 原始文件名
            
        Returns:
            FileStorage 对象
        """
        # 计算MD5
        md5_hash = FileStorage.get_md5(file_obj)
        
        # 检查是否已存在
        existing = FileStorage.objects.filter(md5_hash=md5_hash, 
                                            storage_type=storage_type).first()
        if existing:
            return existing
        
        # 获取存储路径
        relative_path = FileStorage.get_storage_path(md5_hash, storage_type)
        full_path = settings.STORAGE_ROOT / relative_path
        
        # 创建目录
        full_path.parent.mkdir(parents=True, exist_ok=True)
        
        # 保存文件
        file_name = original_name or file_obj.name
        file_path = full_path / file_name
        
        with open(file_path, 'wb+') as destination:
            for chunk in file_obj.chunks():
                destination.write(chunk)
        
        # 保存元信息
        file_storage = FileStorage.objects.create(
            md5_hash=md5_hash,
            file_name=file_name,
            file_size=file_obj.size,
            file_type=file_name.split('.')[-1].lower() if '.' in file_name else '',
            mime_type=file_obj.content_type if hasattr(file_obj, 'content_type') else '',
            storage_type=storage_type,
            relative_path=f"{relative_path}/{file_name}",
            document=document
        )
        
        # 保存info.json
        cls._save_file_info(file_storage, file_path.parent)
        
        return file_storage
    
    @classmethod
    def _save_file_info(cls, file_storage, dir_path):
        """保存文件元信息"""
        import json
        info = {
            'md5': file_storage.md5_hash,
            'file_name': file_storage.file_name,
            'file_size': file_storage.file_size,
            'file_type': file_storage.file_type,
            'created_at': file_storage.created_at.isoformat(),
            'document_id': file_storage.document_id,
        }
        info_path = dir_path / 'info.json'
        with open(info_path, 'w', encoding='utf-8') as f:
            json.dump(info, f, ensure_ascii=False, indent=2)
    
    @classmethod
    def get_file(cls, md5_hash, storage_type='original'):
        """获取文件"""
        file_storage = FileStorage.objects.filter(md5_hash=md5_hash, 
                                                storage_type=storage_type).first()
        if not file_storage:
            return None
        return file_storage
    
    @classmethod
    def delete_file(cls, md5_hash, storage_type='original'):
        """删除文件"""
        file_storage = FileStorage.objects.filter(md5_hash=md5_hash, 
                                                storage_type=storage_type).first()
        if file_storage:
            try:
                # 删除物理文件
                file_path = file_storage.full_path
                if file_path.exists():
                    file_path.unlink()
                
                # 删除info.json
                info_path = file_path.parent / 'info.json'
                if info_path.exists():
                    info_path.unlink()
                
                # 删除数据库记录
                file_storage.delete()
                return True
            except Exception as e:
                print(f"删除文件失败: {e}")
                return False
        return False
```

## 7. 文档转换流程

### 7.1 整体流程

```
用户上传文件
    ↓
保存原始文件到 original/ 目录
    ↓
触发文档转换任务（异步）
    ↓
┌─────────────────────────────────┐
│ 1. 解析文件结构                  │
│ 2. 提取图片和附件                │
│ 3. 转换文档内容为Markdown         │
│ 4. 处理图片链接                   │
└─────────────────────────────────┘
    ↓
保存转换后的文件到 converted/ 目录
    ↓
保存附件到 attachments/ 目录
    ↓
更新数据库记录
```

### 7.2 支持的文档类型

| 文件类型 | 扩展名 | 转换目标 |
|---------|--------|---------|
| Word文档 | .docx, .doc | Markdown + 图片附件 |
| PowerPoint | .pptx, .ppt | Markdown + 图片附件 |
| PDF | .pdf | 纯文本 + 图片附件 |
| 纯文本 | .txt, .md | 直接保存 |
| 图片 | .jpg, .png, .gif | 作为附件 |

### 7.3 Word文档转换详情

以Word文档为例的转换流程：

1. **提取图片**
   - 从Word文档中提取所有图片
   - 保存到attachments/目录
   - 生成图片链接
   - 在Markdown中替换为 `![图片描述](附件链接)`

2. **转换格式**
   - 使用python-docx或mammoth解析文档
   - 处理标题、段落、列表、表格等
   - 转换为标准Markdown格式

3. **生成info.json**
   - 记录转换参数
   - 记录附件清单
   - 记录图片映射关系

### 7.4 使用示例

#### 上传并转换单个文件

```python
from .services import StorageService, DocumentConverter

# 上传文件
def upload_and_convert(request, document_id):
    file = request.FILES['file']
    document = Document.objects.get(id=document_id)
    
    # 1. 保存原始文件
    file_storage = StorageService.save_file(
        file_obj=file,
        storage_type='original',
        document=document,
        original_name=file.name
    )
    
    # 2. 开始转换（异步执行）
    converter = DocumentConverter()
    converter.convert_async(file_storage.id)
    
    return Response({
        'file_id': file_storage.id, 
        'md5': file_storage.md5_hash,
        'status': 'converting'
    })
```
