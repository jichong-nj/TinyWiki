## 1. 登录系统

1. 在浏览器中打开系统地址
2. 使用管理员账号登录
3. 登录后点击左下角的"管理后台"进入管理界面

---

## 2. 管理后台概览

管理后台左侧边栏包含以下功能模块：

- 📚 知识库管理
- 📄 文档管理
- 🔐 权限管理
- ⚙️ 系统设置
- 🏠 切换 Wiki（返回前台）
- 👤 退出登录

## 3. 知识库管理

### 3.1 创建知识库

1. 进入"知识库管理"页面
2. 点击"新建知识库"按钮
3. 填写知识库信息：
   - 知识库名称
   - 描述（可选）
4. 点击"保存"完成创建

### 3.2 编辑和删除知识库

- 点击知识库列表中的"编辑"按钮修改信息
- 点击"删除"按钮删除知识库（⚠️ 谨慎操作，删除后不可恢复）

### 3.3 查看知识库信息

- 列表显示所有知识库的名称、创建时间、更新时间

## 4. 文档管理

### 4.1 选择目录和文件夹

1. 进入"文档管理"页面
2. 在顶部选择要管理的知识库和目录
3. 在左侧文件夹树中选择文件夹

### 4.2 新建文档

1. 点击"新建文档"按钮
2. 填写文档信息：
   - 文档标题
   - 选择目录和文件夹（可选）
3. 在编辑器中编写文档内容（支持 Markdown）
4. 可以选择"保存草稿"或"发布"
   - **草稿**：仅在后台可见
   - **发布**：前台用户可见

### 4.3 编辑和删除文档

- 点击文档列表中的"编辑"按钮修改内容
- 点击"删除"按钮删除文档

### 4.4 导入文档

#### 4.4.1 单个文件导入

1. 点击"导入文件"按钮
2. 选择文件（支持 Word、PPT、PDF、Excel、Markdown、TXT 等格式）
3. 系统会自动将文件转换为 Markdown 格式
4. 可以在导入列表中查看上传进度和状态

#### 4.4.2 ZIP 批量导入

1. 点击"导入 ZIP"按钮
2. 选择 ZIP 文件
3. 系统会自动解析并列出文件
4. 勾选需要导入的文件
5. 点击"开始导入"

#### 4.4.3 支持的文件格式

- **Word 文档** (.docx)
- **PowerPoint 演示** (.pptx)
- **PDF 文档** (.pdf)
  - 可解析文本的 PDF 会提取文本和图片
  - 扫描版 PDF 会将每页转换为图片
- **Excel 表格** (.xlsx)
  - 支持多 sheet，每个 sheet 用一级标题分隔
  - 自动提取内嵌图片
- **Markdown** (.md)
- **文本文件** (.txt)

### 4.5 管理目录和文件夹

#### 4.5.1 管理目录

- 点击"目录管理"标签
- 可以新建、编辑、删除目录

#### 4.5.2 管理文件夹

- 在左侧文件夹树中右键或点击操作按钮
- 可以新建、编辑、删除文件夹

## 5. 权限管理

### 5.1 权限系统说明

系统支持两种角色：

- **知识管理员**：可以管理和查看授权的知识库/目录
- **知识使用人**：仅可以查看授权的知识库/目录

权限支持两种粒度：

- **知识库级别**：授权整个知识库
- **目录级别**：仅授权特定目录

### 5.2 分配权限

1. 进入"权限管理"页面
2. 点击"分配权限"按钮
3. 填写权限信息：
   - 选择用户
   - 选择授权类型（知识库/目录）
   - 选择具体的知识库或目录
   - 选择角色（知识管理员/知识使用人）
4. 点击"保存"

### 5.3 管理权限

- 在权限列表中可以查看所有已分配的权限
- 点击"删除"按钮撤销权限
- **注意**：系统管理员的权限受保护，无法修改或删除

### 5.4 权限继承

- 如果用户拥有知识库级别的权限，则自动拥有该知识库下所有目录的权限
- 目录级别的权限不会自动继承到子目录，需要单独分配

## 6. AI 配置

### 6.1 配置 AI 模型

1. 进入"系统设置"页面
2. 配置文本生成模型：
   - 提供商（OpenAI 兼容）
   - API Key
   - Base URL
   - 模型名称
   - 温度参数
3. 配置 Embedding 模型：
   - 提供商
   - API Key
   - Base URL
   - 模型名称
   - 维度
   - 输入类型
4. 配置 Rerank 模型（可选）

### 6.2 测试 AI 模型

1. 在配置页面点击"测试"按钮
2. 系统会测试连接并显示结果
3. 可以查看测试历史记录

### 6.3 AI 功能说明

- **全文搜索**：使用 PostgreSQL 全文索引 + BM25 算法
- **向量检索**：使用 Embedding 模型将文档向量化，支持语义搜索
- **RAG 问答**：基于检索到的文档内容，使用大语言模型生成回答
- **多轮对话**：支持上下文连续对话

## 7. 用户管理

### 7.1 创建用户

目前用户创建通过 Django Admin 后台或 API 进行：

1. 访问 `http://<服务器IP>:8080/admin/`
2. 登录管理员账号
3. 在 Users 部分添加新用户
4. 设置用户名、邮箱和密码

### 7.2 用户类型

- **超级用户**：拥有所有权限，不受限制
- **员工用户**：可以访问管理后台
- **普通用户**：仅能访问前台和已授权的知识库

## 8. 数据备份和恢复

### 8.1 数据库备份

```bash
# 进入项目目录
cd /path/to/TinyWiki

# 使用 Docker Compose 备份
docker-compose exec db pg_dump -U tinywiki_user tinywiki > backup_$(date +%Y%m%d).sql
```

### 8.2 数据库恢复

```bash
# 恢复数据库
cat backup_20260511.sql | docker-compose exec -T db psql -U tinywiki_user tinywiki
```

### 8.3 文件备份

- 定期备份 `media` 和 `storage` 目录
- 建议使用云存储或外部存储设备

## 9. 常见问题

### Q: 如何添加更多用户？
A: 通过 Django Admin 后台（`/admin/`）添加，或使用 API 接口。

### Q: 文档导入失败怎么办？
A: 检查文件格式是否支持，文件大小是否过大（建议单个文件 < 50MB）。

### Q: AI 功能无法使用？
A: 检查系统设置中的 AI 配置，确认 API Key 和地址正确，并测试连接。

### Q: 如何修改系统名称？
A: 目前需要修改代码配置，后续版本会支持在后台设置。

## 10. 最佳实践

1. **知识库组织**：按业务或部门划分知识库
2. **目录结构**：保持清晰的目录层级，便于管理和查找
3. **权限分配**：遵循最小权限原则，只分配必要的权限
4. **定期备份**：建议每日备份数据库和文件
5. **文档规范**：制定文档编写规范，提高知识库质量
6. **AI 优化**：定期评估 AI 回答质量，调整模型参数
