# TinyWiki 数据库索引分析报告

## 分析依据

基于代码分析，主要查询模式包括：

### 1. 文档状态查询
- `Document.objects.filter(publish_status='published')`
- `Document.objects.filter(analysis_status='pending')`

### 2. 目录/文件夹查询
- `Document.objects.filter(directory_id=directory_id)`
- `Document.objects.filter(folder_id=folder_id)`
- `Document.objects.filter(directory=directory, folder__isnull=True)`

### 3. 全文搜索
- 使用 PostgreSQL 全文索引：`search_vector` 字段
- 查询：`Document.objects.filter(search_vector=search_query)`

### 4. 向量搜索
- `DocumentChunk.objects.filter(document__publish_status='published')`
- 向量相似度查询：`embedding_vector` 字段

### 5. 时间排序
- `order_by('-updated_at', '-created_at')`

### 6. 唯一性检查
- `Document.objects.filter(filename=filename)`

## 推荐索引

### 单列索引
1. **publish_status** - 最频繁的过滤条件
2. **analysis_status** - 状态管理查询
3. **directory_id** - 目录关联查询
4. **folder_id** - 文件夹关联查询
5. **updated_at** - 时间排序
6. **filename** - 唯一性检查

### 复合索引
1. **(publish_status, directory_id)** - 已发布文档按目录查询
2. **(publish_status, folder_id)** - 已发布文档按文件夹查询
3. **(directory_id, folder_id, publish_status)** - 复杂文档树查询

### 向量索引
- **embedding_vector** - pgvector 自动创建 IVFFlat 索引

## 性能提升预期

### 查询类型优化

#### 状态过滤查询
```sql
-- 优化前：全表扫描
SELECT * FROM documents_document WHERE publish_status = 'published';

-- 优化后：索引扫描
-- 索引：idx_document_publish_status
```

#### 复合条件查询
```sql
-- 文档树查询
SELECT * FROM documents_document
WHERE publish_status = 'published' AND directory_id = ?
ORDER BY updated_at DESC;

-- 索引：idx_document_publish_directory + idx_document_updated_at
```

#### 全文搜索
```sql
-- 使用 GIN 索引
SELECT * FROM documents_document
WHERE search_vector @@ to_tsquery('chinese', '查询词');
```

### 索引选择性分析

| 索引字段 | 选择性 | 推荐类型 |
|---------|--------|----------|
| publish_status | 高 (2-3个值) | 复合索引 |
| directory_id | 中等 | 单列 + 复合 |
| folder_id | 高 | 单列 + 复合 |
| filename | 高 | 单列 |
| updated_at | 高 | 单列 |

## 实施建议

### 1. 逐步创建索引
```sql
-- 使用 CONCURRENTLY 避免阻塞
CREATE INDEX CONCURRENTLY IF NOT EXISTS idx_name ON table_name (column);
```

### 2. 监控效果
```sql
-- 查看索引使用情况
SELECT * FROM pg_stat_user_indexes
WHERE tablename LIKE 'documents_%'
ORDER BY idx_scan DESC;
```

### 3. 维护策略
- 定期 REINDEX 碎片化的索引
- 清理未使用的索引
- 监控索引大小和性能

## 注意事项

1. **CONCURRENTLY**: 生产环境必须使用，避免阻塞写入
2. **条件索引**: 对于低选择性字段，使用部分索引
3. **复合索引顺序**: 将最选择性的字段放在前面
4. **监控**: 定期检查索引使用情况和性能影响

## 预期性能提升

- **状态查询**: 10-100倍速度提升
- **文档列表**: 5-20倍速度提升
- **搜索查询**: 显著提升（尤其大数据量）
- **排序查询**: 2-5倍速度提升
