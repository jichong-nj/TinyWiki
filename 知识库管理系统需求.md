# 知识库管理系统 — 需求文档

**版本**: v1.2 | **日期**: 2026-05-09

---

## 1. 核心功能：文件导入与存储

### 1.1 支持格式
系统需支持以下格式的**批量导入**：
- Markdown (`.md`)
- Word (`.docx`)
- PPT (`.pptx`)
- PDF (`.pdf`)

### 1.2 统一落盘标准
- **所有导入文件最终均以 Markdown 格式存储**。
- 系统需在导入时自动完成格式转换（如 docx/pdf/ppt → markdown），保留标题、列表、表格等核心结构。

## 2. 目录结构设计

系统采用**三级目录体系**进行管理：

| 层级 | 名称 | 说明 | 示例 |
|------|------|------|------|
| **第一级** | **知识库 (Knowledge Base)** | 最高层级，代表一个大的知识领域或项目集合 | "企业产品知识库"、"技术文档库" |
| **第二级** | **知识目录 (Directory)** | 知识库下的具体分类/专题，是权限管理的核心单元 | "产品 A"、"产品 B"、"产品 C" |
| **第三级** | **子文件夹 & 文件** | 知识目录下的实际内容组织 | `设计稿/`、`需求文档.md`、`API说明.md` |

---

## 3. 权限管理

### 3.1 核心原则
- **按二级目录（知识目录）分配管理员**。
- 不同目录可由不同的人负责管理，实现**分权治理**。

### 3.2 角色示例
| 角色 | 职责范围 | 权限说明 |
|------|----------|----------|
| **知识库超级管理员** | 整个一级知识库 | 创建/删除二级目录、分配目录管理员 |
| **目录管理员 (如产品经理)** | 指定的二级目录（如"产品 A"） | 管理该目录下所有三级文件夹和文件，不可越权访问其他目录 |
| **普通成员** | 全局或指定范围 | 仅查看/编辑被授权的目录内容 |

## 4. 前端界面设计

### 4.1 顶部导航栏 (Top Panel)

#### 4.1.1 当前知识库上下文展示
- **位置**：页面最顶部，固定显示。
- **功能**：展示当前选中的**一级知识库**。
- **交互**：支持切换不同的知识库（如果用户有多个权限）。

#### 4.1.2 统计面板
在顶部导航栏中实时展示以下统计数据：

| 指标 | 说明 |
|------|------|
| **未发布文档数** | 当前知识库/目录下处于"草稿"状态的文档数量 |
| **待分析文档数** | 已发布但尚未完成 embedding + 全文索引构建的文档数量 |

- **数字可点击**，点击后跳转到对应列表页。
- **列表页功能**：展示该类别下的所有文档，支持批量选择、逐个或批量执行操作（如批量发布、批量触发分析）。
- 数据实时刷新（或近实时更新）。

## 6. 文件编辑与预览

### 6.1 核心流程
1. **查看**：点击文件，根据权限展示 Markdown 渲染后的内容。
2. **进入编辑**：点击"编辑"按钮（无权限时隐藏或置灰）。
3. **编辑器模式**：
   - **左右分屏**：左侧源码 / 右侧实时预览。
   - **全屏编辑**：沉浸式写作，顶部保留工具栏。
   - **纯预览**：仅展示渲染效果。

### 6.2 编辑器选型建议
- 推荐采用成熟方案（如 TipTap, Monaco Editor + MDX, 或 ProseMirror）。
- 支持基础 Markdown 语法（标题、列表、表格、代码块等）。

## 7. 文档状态管理

### 7.1 两个独立维度

每个文档有两个核心状态，分别控制不同的能力：

| 维度 | 状态值 | 含义 |
|------|--------|------|
| **发布状态** | `未发布` / `已发布` | 是否对公开 Wiki 站点可见 |
| **分析状态** | `未分析` / `已分析` | 是否完成了 embedding + 全文索引构建（RAG 知识库） |

### 7.2 状态组合与含义

| 发布状态 | 分析状态 | 可用操作 | Wiki 可见？ | RAG 可用？ |
|----------|----------|---------|------------|-----------|
| ❌ 未发布 | ❌ 未分析 | 编辑、删除 | ❌ | ❌ |
| ✅ 已发布 | ⏳ 待分析 | 编辑、删除 | ✅（暂不可检索） | ⏳ 进行中 |
| ✅ 已发布 | ✅ 已分析 | 编辑、删除 | ✅ | ✅ |

### 7.3 关键约束

1. **必须先发布才能触发分析** — 未发布的文档不会进入分析队列。
2. **已发布不可下架/撤回** — 一旦点击"发布"，该文档永久对外可见，无法撤销。只能编辑内容或删除整个文档。
3. **已分析的文档修改后需重新分析** — 内容变更触发重新构建 embedding + 索引。

### 7.4 状态流转图

```
[未发布/未分析] → (点击"发布") → [已发布/待分析] → (自动进入分析队列) → [已发布/已分析]
                                              ↓
                                    (编辑内容后) → 重新触发分析
                                              ↓
                                    (删除文档) → 从系统中移除
```

## 8. 全文索引系统

### 8.1 功能概述

系统采用 PostgreSQL 全文索引技术，支持中文搜索。全文索引与文档的发布状态绑定，仅对已发布文档建立索引。

### 8.2 索引更新机制

| 场景 | 触发条件 | 索引操作 |
|------|----------|----------|
| **创建索引** | 文档发布（publish_status 变为 'published'） | 自动创建全文索引 |
| **更新索引** | 已发布文档内容被修改并保存 | 自动更新索引 |
| **删除索引** | 文档删除或取消发布 | 自动删除索引 |

### 8.3 索引内容

索引包含以下字段：
- **title**：文档标题
- **content**：文档内容

### 8.4 搜索接口

#### 8.4.1 文档搜索

- **URL**: `GET /api/documents/search/`
- **权限**: 需认证

**请求参数**:

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `q` | string | ✅ | 搜索关键词 |
| `directory` | int | ❌ | 限制搜索范围到指定目录 |
| `folder` | int | ❌ | 限制搜索范围到指定文件夹 |

**响应示例**:
```json
[
  {
    "id": 1,
    "title": "API 接口文档",
    "filename": "api-doc.md",
    "content": "...",
    "publish_status": "published",
    "analysis_status": "completed",
    "directory": 1,
    "folder": null
  }
]
```

## 9. 文档版本管理

### 9.1 核心机制
- **每次编辑保存 → 自动生成新版本号**。
- 版本号格式：`v1, v2, v3...`（从 v1 开始递增）。
- 初始导入/创建时自动生成为 `v1`。

### 8.2 版本信息
每个版本记录以下元数据：

| 字段 | 说明 |
|------|------|
| **版本号** | v1, v2, v3... |
| **修改时间** | 精确到秒 |
| **修改人** | 执行编辑操作的用户 |
| **变更摘要** | （可选）用户填写的修改说明 |

### 8.3 版本历史查看
- 在文档详情页提供**"版本历史"入口**。
- 点击后展示该文档的所有版本列表（按时间倒序排列）。每个版本显示：版本号、修改时间、修改人、变更摘要。

### 8.4 版本对比与回滚
- **版本对比**：选择任意两个版本，系统高亮展示差异内容（diff 视图）。
- **版本回滚**：可将文档恢复到指定历史版本。回滚后生成一个新的版本号（如 v5），保留原始版本记录。

### 8.5 版本与分析的关联
- 已发布且已分析的文档，如果用户修改并保存 → 新版本生成 + **自动触发重新分析**。
- 未发布的草稿文档修改不影响任何外部状态。

## 10. AI 模型基础配置

系统依赖多个 AI 模型完成 RAG 知识库构建与检索、Wiki 站点智能问答等核心能力。所有模型配置集中在后台管理系统的**"AI 模型配置"**页面中统一管理。

### 10.1 文本生成 / 聊天模型（必选）
- **用途**：
  - RAG 知识库构建中的文档摘要、内容理解
  - Wiki 公开站点的智能问答对话（用户提问 → 基于知识库内容的回答）
- **配置项**：

| 配置项 | 说明 |
|--------|------|
| **模型提供商/接口** | OpenAI / Anthropic / 本地部署（如 Ollama）等 |
| **API Key** | 认证密钥 |
| **Base URL** | API 端点地址（本地部署时填写内网地址） |
| **模型名称** | 具体模型 ID，如 `gpt-4o`、`qwen-max`、`claude-sonnet-4-20250514` 等 |
| **Temperature** | 生成随机性参数（默认值可配置） |

### 9.2 Embedding 模型（必选）
- **用途**：将文档内容转换为向量，用于 RAG 知识库的语义检索。
- **配置项**：

| 配置项 | 说明 |
|--------|------|
| **模型提供商/接口** | OpenAI / Cohere / 本地部署（如 Ollama）等 |
| **API Key** | 认证密钥 |
| **Base URL** | API 端点地址 |
| **模型名称** | 具体模型 ID，如 `text-embedding-3-large`、`bge-m3` 等 |
| **向量维度** | 输出向量的 dimension（部分接口需手动指定） |

### 9.3 Rerank 模型（必选）
- **用途**：RAG 检索中对初步召回结果进行重排序，提升回答精准度。
- **配置项**：

| 配置项 | 说明 |
|--------|------|
| **模型提供商/接口** | Cohere / BGE / 本地部署等 |
| **API Key** | 认证密钥 |
| **Base URL** | API 端点地址 |
| **模型名称** | 具体模型 ID，如 `bge-reranker-v2-m3`、`cohere-rerank` 等 |

### 9.4 图文分析模型（可选）
- **用途**：对导入的 PDF/图片类文档进行 OCR + 内容理解，提取其中的文字和结构化信息。
- **关联场景**：在 RAG 知识库构建中用于处理含图表、截图的文档。
- **配置项**：

| 配置项 | 说明 |
|--------|------|
| **模型提供商/接口** | OpenAI GPT-4o / Claude / 本地部署等 |
| **API Key** | 认证密钥 |
| **Base URL** | API 端点地址 |
| **模型名称** | 具体模型 ID，如 `gpt-4o`、`claude-opus-4-20250514` 等 |

### 9.5 配置管理
- **保存与测试**：每项配置保存后可点击"测试连接"验证连通性。
- **多模型支持**：允许为不同知识库/目录配置不同的模型（按优先级：全局 → 知识库级 → 目录级）。
- **敏感信息保护**：API Key 等敏感字段在数据库中加密存储，界面中脱敏显示。

## 11. 公开 Wiki 浏览网站

### 11.1 定位
面向最终读者的**只读站点**，用于浏览和检索已发布的知识库内容。与后台管理系统完全分离。

### 11.2 顶部导航栏

#### 10.2.1 知识库选择器
- **位置**：页面最顶部左侧或居中。
- **功能**：下拉菜单/标签页形式，展示用户有权限访问的所有知识库列表。
- **交互**：点击切换当前浏览的知识库，切换后侧边栏和主内容区同步更新到该知识库的根目录。

#### 10.2.2 搜索框
- **位置**：顶部导航栏右侧。
- **功能**：全局搜索（全文 + RAG 语义检索），支持按当前知识库范围过滤。

#### 10.2.3 智能问答入口（顶部导航栏）
- **位置**：顶部导航栏中，搜索框旁边。
- **交互**：点击后从页面右侧滑出/弹出一个完整的聊天窗口（宽度约 480px），覆盖部分主内容区。再次点击或点击遮罩关闭。

##### 10.2.3.1 知识范围选择器
在聊天窗口顶部提供范围选择控件：

| 层级 | 是否必填 | 说明 |
|------|---------|------|
| **知识库** | ✅ 必选 | 用户选择一个一级知识库，仅在该范围内检索 |
| **知识目录** | ⭕ 可选 | 在选定知识库下进一步选择二级目录，精确到目录级 |

- **默认选中当前知识库**：弹出对话框时，自动预选顶部导航栏中正在浏览的知识库。
- 用户可随时切换为其他知识库或清空目录筛选。
- 范围切换后，后续提问自动应用新范围。

##### 10.2.3.2 对话交互
- **输入框**：用户输入问题，支持多行文本。
- **回答展示**：
  - AI 生成的自然语言回答。
  - **引用来源**：每条回答下方列出引用的文档列表（文件名 + 片段摘要），点击可跳转到原文位置。
- **追问能力**：支持基于当前上下文的多轮对话。

##### 10.2.3.3 RAG 检索流程
```
用户提问 → 根据选定范围过滤文档 → Embedding 语义检索 → Rerank 重排序 → 文本生成回答 + 引用来源
```

### 11.3 知识库内浏览

#### 10.3.1 侧边栏目录树
- **左侧展示**：当前知识库的完整三级目录结构（知识目录 → 子文件夹 → 文件）。
- **展开/折叠**：支持逐级展开与收起，默认展开到当前选中节点。
- **高亮当前项**：正在浏览的文件在目录树中高亮显示。

#### 10.3.2 主内容区（只读浏览）
- **点击文件 → 打开只读页面**，用于浏览该文件的 Markdown 渲染内容。
- **不可编辑**：无编辑入口、无编辑按钮。
- **面包屑导航**：显示当前路径（如 `产品 A / 设计稿 / API说明.md`），支持逐级跳转。

#### 10.3.3 文件内目录导引 (TOC)
- **右侧固定面板**：根据文件的标题层级自动生成 TOC（目录索引）。
- **滚动联动**：浏览内容时，当前所在章节在 TOC 中高亮；点击 TOC 项可平滑跳转到对应位置。
- **展开/收起**：支持折叠整个 TOC 面板以扩大阅读区域。

### 11.4 链接功能

#### 10.4.1 内部文件链接
- **知识库内跳转**：Markdown 中的 `[[文件名]]` 或 `[文本](./其他文件.md)` 链接，点击后在 Wiki 站点内直接跳转到目标文档（无需下载）。
- **同目录/跨目录**：支持同一知识目录下和跨目录的文件互链。

#### 10.4.2 外部链接
- Markdown 中的普通外链（如 `https://...`）正常渲染为可点击超链接，在新标签页中打开。

#### 10.4.3 反向引用 / 相关文档
- **自动关联**：系统根据文件间的链接关系，在页面底部展示"引用此文件的文档"和"被引用的文档"列表。
- **帮助导航**：方便读者发现知识库中的相关内容。
