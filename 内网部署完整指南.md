# TinyWiki 内网离线部署完整指南

## 概述

本指南介绍如何将TinyWiki项目部署到隔离的内网环境中。整个部署过程分为两个阶段：
1. **有网络环境准备阶段**：构建前端、打包镜像和源码
2. **内网部署阶段**：加载镜像并启动服务

## 第一阶段：在有网络的环境中准备部署包

### 步骤1：前端构建

```bash
# 进入前端目录
cd frontend

# 安装依赖
npm install

# 构建生产版本
npm run build

# 构建完成后会生成 dist/ 目录
ls dist/
```

### 步骤2：修改前端API配置（如果需要）

如果内网服务器IP或端口与默认配置不同，需要修改 `frontend/src/axios.ts`：

```typescript
const instance = axios.create({
  baseURL: 'http://your-server-ip:8080/api',  // 修改为实际的服务器IP和端口
  timeout: 120000,
  headers: {
    'Content-Type': 'application/json'
  }
})
```

### 步骤3：重新构建前端（如果修改了配置）

```bash
cd frontend
npm run build
```

### 步骤4：构建Docker镜像

```bash
# 在项目根目录
cd /home/jichong/Projects/TinyWiki

# 构建Web应用镜像
docker build -t tinywiki-web:latest .

# 构建数据库镜像
docker build -t tinywiki-db:latest ./postgres
```

### 步骤5：打包镜像和源码

```bash
# 保存所有镜像
docker save -o tinywiki-images.tar tinywiki-web:latest tinywiki-db:latest nginx:alpine

# 打包源码和配置文件
tar -czf tinywiki-deploy.tar.gz \
    knowledge_base/ \
    frontend/dist/ \
    nginx.conf \
    docker-compose.prod.yml \
    requirements.txt \
    postgres/initdb/ \
    docs/

# 创建部署说明
cat > DEPLOY_README.md << 'EOF'
# TinyWiki 内网部署说明

## 部署前准备
1. 确保目标服务器已安装 Docker 和 Docker Compose
2. 确保服务器有足够的磁盘空间（推荐至少 10GB）
3. 确保端口 80 和 8080 未被占用

## 部署步骤

### 1. 解压部署包
```bash
tar -xzf tinywiki-deploy.tar.gz
cd tinywiki-deploy
```

### 2. 加载Docker镜像
```bash
docker load -i ../tinywiki-images.tar
```

### 3. 启动服务
```bash
docker compose -f docker-compose.prod.yml up -d
```

### 4. 等待服务启动
```bash
# 等待约30秒让服务完全启动
docker compose -f docker-compose.prod.yml ps
```

### 5. 创建管理员账户（可选）
```bash
docker compose -f docker-compose.prod.yml exec web python manage.py createsuperuser
```

## 访问应用

- **前端界面**：http://服务器IP
- **后端API**：http://服务器IP:8080/api
- **Django管理后台**：http://服务器IP:8080/admin

## 常用维护命令

```bash
# 查看服务状态
docker compose -f docker-compose.prod.yml ps

# 查看日志
docker compose -f docker-compose.prod.yml logs -f

# 重启服务
docker compose -f docker-compose.prod.yml restart

# 停止服务
docker compose -f docker-compose.prod.yml down

# 备份数据（如果需要）
docker compose -f docker-compose.prod.yml exec db pg_dump -U tinywiki_user tinywiki > backup.sql
```

## 故障排除

### 服务启动失败
```bash
# 检查端口占用
netstat -tlnp | grep -E ':(80|8080|5432)'

# 检查Docker服务状态
systemctl status docker

# 查看详细日志
docker compose -f docker-compose.prod.yml logs
```

### 数据库连接失败
```bash
# 检查数据库服务
docker compose -f docker-compose.prod.yml exec db pg_isready -U tinywiki_user -d tinywiki

# 重新初始化数据库
docker compose -f docker-compose.prod.yml exec web python manage.py migrate
```

### 前端无法访问
- 检查nginx配置是否正确
- 确认前端dist文件是否正确挂载
- 检查防火墙设置
EOF

echo "部署包准备完成！"
echo "生成的文件："
echo "- tinywiki-images.tar (Docker镜像)"
echo "- tinywiki-deploy.tar.gz (源码和配置)"
echo "- DEPLOY_README.md (部署说明)"
```

## 第二阶段：内网部署

### 传输文件到内网服务器

将以下文件传输到内网服务器：
- `tinywiki-images.tar`
- `tinywiki-deploy.tar.gz`
- `DEPLOY_README.md`

### 在内网服务器上执行部署

按照 `DEPLOY_README.md` 中的步骤执行即可。

## 架构说明

部署后的架构如下：

```
Internet/Intranet
        ↓
    Nginx (端口80)
    ├── 前端静态文件 (frontend/dist)
    └── API代理 → Gunicorn (端口8080)
                   ↓
               Django应用
                   ↓
            PostgreSQL (端口5432)
```

## 安全注意事项

1. **修改默认密码**：部署后立即修改数据库密码和Django SECRET_KEY
2. **网络隔离**：确保只有内网可以访问相关端口
3. **定期备份**：设置定期备份数据库和上传的文件
4. **日志监控**：定期检查应用日志，及时发现异常

## 性能优化建议

1. **数据库索引**：运行项目中的 `create_indexes.sql` 创建索引
2. **静态文件缓存**：nginx配置中已设置静态文件缓存
3. **Gunicorn workers**：根据服务器CPU核心数调整worker数量
4. **数据库连接池**：生产环境建议配置数据库连接池

## 更新部署

当需要更新应用时：
1. 在有网络环境重新构建镜像
2. 打包新的镜像和源码
3. 在内网停止服务、加载新镜像、启动服务</content>
